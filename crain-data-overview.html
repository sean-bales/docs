Make data trustworthy. Make the warehouse the source. Make it AI-ready.

Three destinations. Today’s stack hits none; tomorrow’s hits all.

01Today’s stack

The data flows. The architecture connecting it doesn’t.

Four upstream surfaces. No contract. Tableau reads the result.

Each layer maintained by different people on different cadences. No automated test asserts the four agree on what a field means.
  • Four config surfaces, four cadences and no data contract between them. Names drift. eVars carry labels that no longer match their content.
  • Adobe Launch config schema drift across brand websites, no single schema.
  • 28 Redshift columns silently dropped 2+ years of Adobe text content at load. Data intact in S3. Fix is mechanical.
  • ~175 data elements configured in Adobe Launch; only 25 of those reach the warehouse. 31 of 49 active Adobe variables unread by any dbt model (they sit unused in the raw Adobe tables).
  • 50–75% of Adobe Analytics data being captured is noise, dead, or redundant. Executive reporting already runs on Pelcro and Redshift, not Adobe directly.

Source: cdw-dbt knowledge substrate — Phases 0–8 + Audits 1–3. Compiled 04/27/2026. Universe: 7-system Crain web data estate (data layer, Launch, Analytics, Pelcro, Redshift raw / staging / presentation). Per-system depth: knowledge/.

02Tomorrow’s stack

Warehouse native AI-ready data.

Contract anchors the stack.

  • Website data contract anchors the stack. Schema and test guardrails in code. Every layer downstream consumes the same shape.
  • Warehouse-native pipeline. Website data → API Gateway → Redshift as system of record. AWS infrastructure end to end.
  • Business Insights via AI reading the canonical Redshift store. Replaces dashboards as the primary reporting surface.
  • Kill first. Retire what isn’t read before adding anything new. Phased dual-publish — one metric at a time, retire the legacy version on a published date.
  • Adobe Analytics and Launch retire. Low-to-mid six-figure annual run-rate savings. One-time implementation in same range. Adobe contract renewal is the natural decision point.

Source: working hypothesis derived from cdw-dbt knowledge substrate. Compiled 04/27/2026. Savings figures: rough estimate, assumptive math; firm numbers depend on Adobe contract terms. Discovery work (stakeholder interviews, kill list, metric list) precedes build.
