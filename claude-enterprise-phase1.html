A3Skills management

Skills flow through a Git + PR workflow. Engineering reviews; Help Desk admin-uploads.

Authoring

Builders (the 12 P1 Builder seats) author skills. Users only build personal skills, not org-shared. Authors work in claude.ai's personal skill space or locally as SKILL.md + resources, then submit to Git for review. Frontmatter requires name (lowercase / digits / hyphens, max 64 chars) and description (third-person, what + when, max 1024 chars).

Distribution & versioning

Repository: crain-claude-skills in the Crain GitHub org. Authors open PRs with eval queries (trigger / no-trigger / ambiguous) and a risk-tier declaration. On merge, a manual admin-upload pushes the skill to the Anthropic org — no public API for org provisioning. Git is the system of record; rollback = re-upload prior version. Skills do not sync across surfaces (claude.ai, Claude Code, API) — each is a separate upload.

Governance — risk tiers

Tier Includes Reviewer
LowPrompt-only. No code, MCP, network, credentials, or filesystem writes.Engineering approves.
MediumMCP refs, network calls, filesystem reads.Engineering reviews with added technical scrutiny.
HighCode execution, credentials, filesystem writes, agentic behavior.Engineering reviews technical and policy before admin upload.

Org-wide skill sharing stays disabled at launch — all org skills are admin-uploaded after PR review. Personal skills and "shared with you" peer skills are allowed but don't propagate to the org.
